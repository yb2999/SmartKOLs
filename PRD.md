# SmartKOLs 产品需求文档（PRD）

**版本**：v0.2
**日期**：2026-04-10
**状态**：Demo 阶段（前端原型完成）

---

## 1. 产品概述

SmartKOLs 是一款面向 Twitter 的 AI KOL 矩阵管理工具，提供网页版操作看板。用户可批量管理数十至数百个 Twitter 账号，为每个账号配置独立的 AI 人格、信息源和自动发帖策略，并通过统一的监控中心跟进合作邀约等私信消息。

当前版本为前端 Demo，全部数据通过 Mock Store 驱动（200 个预置账号），用于验证产品逻辑和交互流程。

---

## 2. 目标用户

| 用户类型 | 描述 |
|---|---|
| 自用场景 | 构建 KOL 矩阵，用于项目推广、话题引导 |
| 商业客户 | KOL 经纪人或机构，需批量管理旗下多个 Twitter 账号 |

---

## 3. 信息架构与导航

系统采用左侧导航 + 主内容区的布局，共 4 个一级入口：

| 模块 | 路由 | 说明 |
|---|---|---|
| 概览 | `/dashboard` | 整体运营数据看板 |
| 账号管理 | `/accounts` | 账号列表、分组、批量操作 |
| 监控中心 | `/monitoring` | 私信监控与分类、报警配置 |
| 设置 | `/settings` | AI 模型配置、通用设置 |

单个账号详情页位于 `/accounts/[id]`，包含 4 个子标签：

| 子页 | 路由 | 说明 |
|---|---|---|
| 人格配置 | `/accounts/[id]/persona` | 该账号的 AI 人格设定 |
| 信息源 | `/accounts/[id]/sources` | 内容来源监控配置 |
| 自动发帖 | `/accounts/[id]/autopost` | 发帖时间、频率、风格 |
| 推文预览 | `/accounts/[id]/preview` | AI 生成推文预览 |

## 4. 核心功能

### 4.1 概览看板（Dashboard）

顶部 4 个统计卡片：

- 总账号数（含运行中数量）
- 总粉丝数（跨所有账号累加）
- 未读消息数（含合作邀约子统计）
- 账号分组数

下方两列展示：

- **账号列表预览**：展示前 5 个账号，含头像、昵称、分组标签、运行状态
- **最新消息预览**：展示前 5 条监控消息，含分类图标与未读标识

### 4.2 账号管理（核心模块）

#### 4.2.1 账号列表
- 支持添加单个账号、批量导入（原 "导入 CSV" 改为 "批量导入" 按钮）
- 表格视图：账号信息、所属分组、运行状态、粉丝数、推文数、管理入口
- 搜索：按 handle 或显示名称筛选
- 左侧分组侧边栏：快速切换筛选分组，显示各分组账号数量

#### 4.2.2 账号分组
预置 5 个分组，每组带颜色标识：

| 分组 ID | 名称 | 颜色 | 账号数量 |
|---|---|---|---|
| grp_001 | 加密KOL组 | `#C8922A` | 50 |
| grp_002 | 科技KOL组 | `#4A9EDB` | 50 |
| grp_003 | AI KOL组 | `#4CAF7D` | 50 |
| grp_004 | 财经KOL组 | `#E05252` | 30 |
| grp_005 | 网红博主组 | `#9B59B6` | 20 |

- 支持新建自定义分组
- 支持"未分组"筛选
- 当前 Demo 共 200 个 Mock 账号

#### 4.2.3 批量操作
选中一个或多个账号后，顶部出现批量操作栏：

- **套用人格模板**：从预置模板库选一套应用到所有选中账号
- **随机生成人格**（新增）：从预定义兴趣池（20 条）、性格池（15 条）、写作风格池（10 条）中随机组合，为每个选中账号生成独立人格
- **移入分组**：批量改变分组归属
- **删除**：批量删除账号

### 4.3 人格配置

#### 4.3.1 手动配置
每个账号可独立配置以下人格字段：

| 字段 | 说明 |
|---|---|
| 性别 | 男 / 女 / 非二元 |
| 国籍 | 角色所在国家（自由文本） |
| 年龄 | 数字 |
| 兴趣爱好 | 标签输入，Enter 添加 |
| 性格特征 | 标签输入 |
| 写作风格 | 文本描述 |
| 个人简介 | Twitter Bio 文案 |

#### 4.3.2 人格蒸馏
- 用户粘贴 5-20 条该账号的已有推文
- AI 自动分析并提取人格特征，填充表单
- 适用于已有真实账号需要还原风格的场景
- 当前为 Mock 实现，后续对接 Claude API

#### 4.3.3 人格模板库
预置多套人格模板（加密分析师、科技博主等），用户可一键套用至单个账号或批量应用。

### 4.4 信息源管理

每个账号可单独配置信息源，AI 根据源内容生成推文。

#### 4.4.1 支持的信息源类型

| 类型 | 图标 | 说明 |
|---|---|---|
| RSS | 📡 | 订阅新闻/博客 RSS Feed |
| 网站 | 🌐 | 监控网站最新内容 |
| Twitter | @ | 追踪特定 Twitter 账号动态 |
| YouTube | ▶️ | 监控 YouTube 频道 |
| Substack | 📖 | 订阅 Substack 专栏 |
| Telegram | ✈️ | 监控 Telegram 频道 |

#### 4.4.2 预置信息源库
按领域分类，一键添加：

- **加密 / Web3**：CoinDesk、CoinTelegraph、The Block、Decrypt
- **科技**：TechCrunch、Hacker News、The Verge、Product Hunt
- **AI / 研究**：OpenAI Blog、Anthropic Blog、Alignment Forum、arXiv CS.AI

#### 4.4.3 自定义添加
支持用户输入自定义名称、URL、类型添加任意信息源。信息源卡片展示 favicon、URL、类型标签、启用开关和删除按钮。

### 4.5 自动发帖配置

每个账号可独立配置发帖策略：

| 配置项 | 可选值 |
|---|---|
| 发帖开关 | 开 / 关 |
| 发帖风格 | 随意 / 专业 / 幽默 / 思考型 / 大胆 / 学术 |
| 发帖频率 | 每天 1 次 / 3 次 / 5 次 |
| 发帖时间 | 每个频率位对应一个具体时间点 |
| 活跃天数 | 周一 ~ 周日任选 |
| 发帖话题 | 标签输入，聚焦内容 |
| 避免话题 | 标签输入，排除内容 |
| 包含 Hashtag | 开 / 关 |
| 包含表情符号 | 开 / 关 |

### 4.6 AI 推文预览

- 基于账号人格 + 信息源 + 发帖配置，由 AI 生成推文草稿
- 以 Twitter 风格卡片展示（含头像、昵称、时间戳、互动按钮占位）
- 支持一键"重新生成"和未来的"审核并排期"
- 当前为 Mock 实现，后续对接 Claude API

### 4.7 监控中心

#### 4.7.1 私信监控
自动抓取各 KOL 账号收到的私信，并分类展示：

| 类别 | 标签 | 说明 |
|---|---|---|
| collab | 🤝 合作邀约 | 品牌合作、付费推广邀请 |
| commerce | 💰 商务询价 | 报价、合作条款咨询 |
| spam | 🚨 垃圾信息 | 垃圾/诈骗消息 |
| normal | 💬 普通消息 | 其他日常消息 |

- 左侧消息列表：支持按分类筛选，未读消息有左边框高亮
- 右侧消息详情：展示完整内容，合作/商务类可"回复"或"标记已处理"
- 消息列表顶部显示未读数总计

#### 4.7.2 报警配置
支持将关键私信（如合作邀约）实时推送到外部通道：

- **Lark（飞书）**：通过 Webhook 推送
- **Telegram**：通过 Bot Token + Chat ID 推送
- 触发条件：仅合作邀约 / 所有消息
- 支持"测试发送"验证配置

### 4.8 设置

| 配置项 | 说明 |
|---|---|
| Claude API Key | 用于人格蒸馏和推文生成（密码输入） |
| 模型选择 | Claude Sonnet（推荐） / Claude Haiku（快速） |
| 自动保存配置 | 修改人格/发帖配置时自动保存 |
| 界面语言 | 中文 / English |

## 5. 视觉设计

### 5.1 配色系统（OKX 白色亮色模式）

| 用途 | 颜色 |
|---|---|
| 页面背景 | `#F7F7F7` |
| 卡片/侧边栏背景 | `#FFFFFF` |
| 边框 | `#E8E8E8` |
| 主文字 | `#111111` |
| 次要文字 | `#999999` |

| 激活态背景 | `rgba(0,0,0,0.04)` |
| 成功色 | `#00C087` / `#00BA7C` |
| 警告/危险色 | `#E05252` |

### 5.2 头像方案
账号头像通过 `https://unavatar.io/twitter/{handle}` 拉取真实 Twitter 头像，`avatarSeed` 字段存储对应 Twitter handle。加载失败时自动降级为 DiceBear 随机头像占位。

### 5.3 UI 语言
默认中文。周几、性别选项等均为中文展示。预留英文切换入口。

## 7. 用户主流程

```
登录
 └── 概览看板（查看整体数据）
       │
       ├── 账号管理
       │    ├── 按分组筛选 / 搜索
       │    ├── 新增或批量导入账号
       │    ├── 批量操作：套模板 / 随机人格 / 移分组 / 删除
       │    └── 进入单个账号
       │          ├── 配置人格（手填 / 蒸馏 / 模板）
       │          ├── 配置信息源（预置库 / 自定义）
       │          ├── 配置自动发帖（风格 / 频率 / 时间 / 话题）
       │          └── AI 推文预览与重生成
       │
       ├── 监控中心
       │    ├── 查看分类私信
       │    ├── 处理合作邀约
       │    └── 配置 Lark / Telegram 报警
       │
       └── 设置
            └── 填入 Claude API Key、选择模型
```

## 9. 技术方向

### 9.1 当前技术栈（Demo）
- **前端框架**：Next.js 14（App Router）+ TypeScript
- **样式**：Tailwind CSS（自定义 hex token）
- **UI 组件**：shadcn/ui 风格自建组件 + Radix UI 原语
- **图标**：lucide-react
- **状态管理**：React Context（`MockStoreProvider`）
- **Mock 数据**：`src/data/*.json`（accounts、groups、personas、persona-templates、autopost、sources、tweet-previews、monitoring）

### 9.2 后续集成方向
- **AI 能力**：接入 Claude API 实现真实人格蒸馏、推文生成、私信分类
- **Twitter 接入**：通过 Twitter API 实现账号信息拉取、发帖、回复、私信抓取
- **持久化存储**：从 Mock JSON 替换为数据库（账号信息、人格配置、发帖记录、监控消息）
- **外部通知**：对接 Lark Webhook 与 Telegram Bot API
- **用户系统**：登录鉴权、订阅计费、多账号权限
