# SmartKOLs 产品需求文档（PRD）

**版本**：v0.4
**日期**：2026-04-10
**状态**：Demo 阶段（前端原型完成，已部署至 https://smartkols.vercel.app/）

---

## 1. 产品概述

SmartKOLs 是一款面向 Twitter 的 AI KOL 矩阵管理工具，提供网页版操作看板。用户可批量管理数十至数百个 Twitter 账号，为每个账号配置独立的 AI 人格、信息源与自动发帖策略，并通过自动互动（关注、转发、评论、回复）让账号看起来像真人，而不是只会发推的机器。

当前版本为前端 Demo，全部数据通过 Mock Store + localStorage 伪持久化驱动（200 个预置账号），用于验证产品逻辑与交互流程。首次访问会自动触发 2 分钟的应用内产品导览，引导访客理解核心价值主张。支持移动端浏览。

|---|---|

| 商业客户 | KOL 经纪人或机构，需批量管理旗下多个 Twitter 账号 |

## 3. 信息架构与导航

系统采用左侧导航 + 主内容区布局，共 **6 个一级入口**：

| 模块 | 路由 | 说明 |
|---|---|---|
| 概览 | `/dashboard` | 运营数据看板 + 热门话题 |
| 账号管理 | `/accounts` | 账号列表、分组、批量操作、健康评分 |
| 内容日历 | `/calendar` | 整个矩阵的发帖排期周视图 |
| 内容审核 | `/drafts` | AI 草稿待批准队列 |
| 监控中心 | `/monitoring` | 私信监控 + 分类 + 报警 |
| 设置 | `/settings` | AI 配置、团队成员、账户管理 |

此外还有：
- **登录页** `/login` —— 首页门面
- **全局搜索** Cmd+K 命令面板（任意页面可用）
- **通知中心** 侧边栏铃铛入口
- **产品导览** 侧边栏底部"产品导览"按钮，首次访问自动提示

账号详情页位于 `/accounts/[id]`，包含 **6 个子标签**：

| 子页 | 路由 | 说明 |
|---|---|---|
| 人格配置 | `/persona` | AI 人格设定 + 健康分卡片 |
| 信息源 | `/sources` | 内容来源监控配置 |
| 自动发帖 | `/autopost` | 发帖时间、频率、风格 |
| **互动自动化** | `/engagement` | 自动关注/转发/评论/回复 + 互动日志 |
| 推文预览 | `/preview` | AI 生成推文预览 |
| **数据分析** | `/analytics` | 粉丝增长、互动统计、热力图 |

## 4. 核心功能

### 4.1 概览看板（Dashboard）

**6 个统计卡片**：总账号数 · 总粉丝数 · 待审核草稿 · 今日互动 · 未读消息 · 账号分组。

**热门话题面板**：展示近 24 小时 10 个热门话题（分"加密/AI/财经"等类别，带热度数值）。点击任一话题 → 一键为矩阵批量生成相关推文草稿 → 跳转到内容审核页。

**账号列表预览** + **最新消息预览**：Dashboard 下方并排显示前 5 项。

### 4.2 账号管理

#### 4.2.1 账号列表
- 支持添加单个账号、批量导入
- 表格视图：账号信息、分组、运行状态、**健康分**、粉丝数、推文数、管理入口
- 搜索与分组侧边栏筛选

#### 4.2.2 账号分组
5 个预置分组：加密KOL组 50 / 科技KOL组 50 / AI KOL组 50 / 财经KOL组 30 / 网红博主组 20。

#### 4.2.3 批量操作
选中后顶部出现操作栏：
- 套用人格模板
- **随机生成人格**（从 20 条兴趣池 / 15 条性格池 / 10 条写作风格池中随机组合）
- 移入分组
- 删除

### 4.3 人格配置

#### 4.3.1 手动配置
字段：性别 / 国籍 / 年龄 / 兴趣爱好 / 性格特征 / 写作风格 / 个人简介。

#### 4.3.2 人格蒸馏
粘贴 5-20 条真实推文 → AI 提取人格特征并填充表单（当前为 Mock，后续对接 Claude API）。

#### 4.3.3 人格模板库
预置多套模板（加密分析师、科技博主等），一键套用。

### 4.4 信息源管理

支持 6 种类型：RSS / 网站 / Twitter / YouTube / Substack / Telegram。

预置库分 3 类：加密/Web3 · 科技 · AI/研究，共 12 个内置源。支持自定义 URL 添加。

### 4.5 自动发帖配置

| 配置项 | 可选值 |
|---|---|
| 发帖开关 | 开/关 |
| 发帖风格 | 随意/专业/幽默/思考型/大胆/学术 |
| 频率 | 每天 1/3/5 次 |
| 发帖时间 | 每个频率位一个具体时间点 |
| 活跃天数 | 周一 ~ 周日任选 |
| 发帖话题 / 避免话题 | 标签输入 |
| Hashtag / 表情符号 | 开/关 |

### 4.6 内容审核（草稿队列）

**入口**：一级导航"内容审核"，Dashboard 的"待审核草稿"统计卡片。

所有 AI 生成的推文进入"待审核"列表。每条可：
- ✅ **批准** —— 进入已批准状态，同步出现在内容日历中
- ✏️ **编辑** —— 内联修改后保存
- 🔄 **重新生成** —— AI 产出新版本
- ❌ **拒绝** —— 标记为拒绝

按状态筛选 tabs：全部 / 待审核 / 已批准 / 已拒绝。

### 4.7 内容日历

**周视图**：7 天 × 账号 × 时间点网格，显示整个矩阵的发帖排期。点击某一天 → 下方展开完整排期详情（时间 + 账号头像 + 话题 + 内容）。

**数据来源**：每个账号的 autopost 配置（activeDays × scheduledTimes）+ 已批准的草稿。

### 4.8 社交互动自动化 ⭐

让每个账号不只是发推，还会主动关注、转发、评论、回复 —— 看起来像真人而非机器。

#### 4.8.1 自动关注 Auto-Follow
- 规则：按关键词 / 按目标账号的粉丝 / 按话题标签
- 每日上限（防触发封号风控）

#### 4.8.2 自动转发 Auto-Retweet
- 触发条件：最小点赞数 + 白名单账号 + 关键词匹配
- **延迟随机化**：不立即转发，30 分钟~2 小时延迟模拟真人
- 可选 Quote Tweet（转发时带 AI 生成的评论）

#### 4.8.3 自动评论 Auto-Comment
- 目标账号列表
- 评论风格：支持型 / 提问型 / 补充观点型
- 模式：只评最新一条 / 随机选一条

#### 4.8.4 自动回复粉丝 Auto-Reply
- 触发类型：被 @ 提及 / 推文下评论 / 私信
- 规则：只回复关注者 / 白名单关键词
- 回复风格：感谢型 / 互动型 / 简短确认

#### 4.8.5 互动日志
账号详情页右侧展示近 7 天的所有自动互动事件（时间倒序）：
> "2 小时前 · 自动关注了 @BitMEXResearch"
> "3 小时前 · 转发了 @VitalikButerin 的推文"
> "5 小时前 · 评论了 @cz_binance："同意，链上数据印证了这点""
> "6 小时前 · 回复了粉丝 @john_doe 的评论"

**顶部统计卡片**：今日关注 / 今日转发 / 今日评论 / 今日回复 四项实时数据。

### 4.9 数据分析（per 账号）

**粉丝增长折线图**：近 30 天趋势，带渐变填充（SVG 纯手绘，数据由 accountId hash 确定性生成）。

**4 项互动统计**：粉丝增长 / 总点赞 / 总转发 / 总评论。

**最佳发帖时间热力图**：7 × 24 格，颜色深浅代表该时段互动率。

**Top 5 高互动推文**：按点赞/转发/评论综合排名。

### 4.10 账号健康评分

**0-100 综合健康分**，基于 4 个维度（各 25 分）：
- 发帖频率稳定性
- 互动率
- 内容一致性
- 风险信号

按分数分为 3 档：绿色健康（≥80）/ 黄色注意（60-79）/ 红色风险（<60）。

**展示位置**：
- 账号列表表格中显示紧凑彩色徽章
- 人格配置页右侧显示完整卡片（含各维度进度条 + 风险建议）

### 4.11 监控中心

#### 4.11.1 私信监控 + 4 类自动分类
- 🤝 合作邀约
- 💰 商务询价
- 🚨 垃圾信息
- 💬 普通消息

左侧列表按分类筛选，右侧详情展示完整内容与回复按钮。

#### 4.11.2 报警配置
支持 Lark（飞书） 和 Telegram 两种外部通道。配置 Webhook/Bot Token，设置触发条件（仅合作邀约 / 所有消息），支持测试发送。

### 4.12 通知中心

Sidebar 铃铛入口，点开弹出通知面板，时间倒序显示 5 类事件：
- 📝 发帖事件
- 💬 消息事件
- ⚠️ 账号健康异常
- ⚡ 系统操作（批量操作、草稿生成等）
- 🤝 互动事件（自动关注/转发/评论/回复）

未读消息有红色标记，支持"全部标为已读"。

### 4.13 全局搜索（Cmd+K 命令面板）

按 **⌘K** 或 **Ctrl+K** 弹出命令面板，模糊搜索：
- 6 个一级页面（直接跳转）
- 200 个账号（按 handle / displayName）
- 推文草稿（按内容 / 话题）
- 监控消息（按发件人 / 正文）

支持 ↑↓ 方向键导航、Enter 确认、ESC 关闭。

### 4.14 登录与身份

#### 4.14.1 登录页 `/login`
- 左侧品牌介绍（4 项核心价值点）
- 右侧登录按钮：GitHub / Google（内嵌官方 SVG 图标）+ "直接体验 Demo"
- Mock 登录：点击任意按钮 → 写入 localStorage → 跳转 Dashboard

#### 4.14.2 团队协作
Settings 页内显示团队成员列表（5 位 mock 成员，含头像/邮箱/角色/最后活跃时间）。角色：Owner / Admin / Editor / Viewer。"邀请成员"按钮占位。

#### 4.14.3 账户操作
- **重置 Demo 数据**：清除所有 localStorage 修改，恢复到初始 Mock
- **退出登录**：清除登录状态，返回登录页

| 模型选择 | Claude Sonnet / Haiku |
| 自动保存配置 | 开关 |
| 界面语言 | 中文 / English |

### 4.16 产品导览（Product Tour）

应用内引导式导览，用**叙事贯穿所有功能**，让访客在 2-3 分钟内理解产品核心价值主张，而非逐个菜单说明。

#### 4.16.1 入口
- **首次访问 Dashboard 自动提示**：黑色渐变欢迎浮卡"第一次访问 SmartKOLs？"，带"开始导览"和"下次再说"按钮。localStorage 记录 `smartkols_tour_seen`，之后不再自动弹出。
- **侧边栏常驻按钮**：`产品导览` 按钮（带 2 min 标签），任意时候可手动触发。
- **重置 Demo 数据** 会一起清掉 `tour_seen` 标记。

#### 4.16.2 叙事设计理念
遵循三个"要"：
- **每一步回答一个"为什么"** —— 不是"这是 X 功能"，而是"为什么你需要 X"
- **反差与意外** —— 用"你以为... 其实..."的结构制造亮点
- **数字是武器** —— 具体数字让信息立体（"200 账号"、"近 7 天 50 条互动"、"延迟 30 分钟~2 小时"）

#### 4.16.3 12 步流程
| # | 页面 | 叙事要点 |
|---|---|---|
| 1 | `/dashboard` | 欢迎 —— "一个人管理 200 个 Twitter KOL 账号" |
| 2 | `/dashboard` stats | 运营心跳 —— 6 张卡片压缩全局状态 |
| 3 | `/dashboard` trending | AI 杠杆 —— 点话题一键生成草稿 |
| 4 | `/accounts` 分组栏 | 规模化秩序 —— 200 账号跨 5 分组 |
| 5 | `/accounts` 健康列 | 防封号预警 —— 直击"半夜醒来账号全挂了"的恐惧 |
| 6 | `/accounts/acc_001/persona` | 独立人格 —— 200 套配置不是复制粘贴 |
| 7 | `/accounts/acc_001/engagement` ⭐ | **灵魂亮点** —— 让账号活起来：关注/转发/评论/回复 |
| 8 | 同上（日志面板） | 透明可审计 —— 所有自动行为都有日志 |
| 9 | `/drafts` | 内容生产线 —— AI 生产，你审核 |
| 10 | `/calendar` | 全局排期 —— 一眼看整个矩阵的发帖计划 |
| 11 | Sidebar Cmd+K | 专业感 —— 全局搜索 |
| 12 | `/dashboard` | 闭幕 —— 鼓励自由探索 |

#### 4.16.4 交互细节
- **SVG 镂空高亮**：目标元素周围全屏暗化，目标区"透出来"
- **自动定位 Tooltip**：根据目标元素在视口的位置，自动选择下/上/右/左
- **跨路由同步**：next 按钮自动 `router.push` 到下一步的路由，等 DOM 渲染后再定位
- **键盘支持**：← → 翻页、ESC 退出
- **元素兜底**：目标元素 selector 未找到时重试 3 次，最终退化到居中显示
- **移动端**：tooltip 自动切换为底部固定全宽抽屉

#### 4.16.5 为什么自研而不用 driver.js / intro.js
现成库都是单页应用设计，跨路由切换时 DOM 卸载会导致高亮框位置错乱。本 demo 需要在 6 个不同路由之间跳转，必须自己写路由感知逻辑。另外自研可以完全复用现有白色模式设计 token，tooltip 和产品浑然一体。

### 5.1 配色系统（OKX 白色亮色模式）

| 用途 | 颜色 |
|---|---|
| 页面背景 | `#F7F7F7` |
| 卡片/侧边栏 | `#FFFFFF` |
| 边框 | `#E8E8E8` |
| 主文字 | `#111111` |
| 次要文字 | `#999999` |
| 强调/按钮 | `#111111` |
| 成功色 | `#00BA7C` |
| 警告/危险 | `#E05252` |

### 5.2 头像方案
账号头像通过 `https://unavatar.io/twitter/{handle}` 拉取真实 Twitter 头像，失败时自动降级为 DiceBear 随机头像。

### 5.3 持久化
所有用户操作（随机人格、新增账号、批准草稿、互动配置等）通过 localStorage 伪持久化，刷新不丢失。设置页提供"重置 Demo 数据"按钮。

### 5.4 语言
默认中文，预留英文切换入口。

### 5.5 响应式适配

支持移动端浏览，**断点 768px**（Tailwind `md`）：

| 组件/页面 | 桌面端（≥768px） | 移动端（<768px） |
|---|---|---|
| Sidebar | 左侧固定 224px | 隐藏，改为顶部 56px 导航栏（汉堡菜单 + Logo + 搜索图标），汉堡触发左侧滑入抽屉 + 遮罩 |
| 主内容区 | `ml-56` | `ml-0 pt-14` |
| Dashboard 统计 | 6 列 | 2 列 |
| 账号列表 | 左侧分组栏 + 表格 | 分组栏隐藏，顶部 `<select>` 下拉筛选；表格 `min-w-[760px]` 横向滚动；按钮只显示图标 |
| 账号详情 sticky 头部 | `top-0` | `top-14`（避开移动端顶部导航） |
| 账号详情 6 tabs | 横排 | 横向滚动（`overflow-x-auto`） |
| 互动自动化 4 统计 | 4 列 | 2 列 |
| 数据分析 4 统计 | 4 列 | 2 列 |
| 数据分析热力图 | 直接渲染 | 横向滚动 |
| 内容日历 7 天视图 | `grid-cols-7` 并排 | `grid-cols-1` 纵向列表 |
| 监控中心 | 左右分栏（消息列表 + 详情） | 上下堆叠（列表 `max-h-[50vh]`，详情在下） |
| 产品导览 Tooltip | 浮动定位（下/上/右/左） | 底部固定全宽抽屉 |
| 登录页 | 左右分栏（品牌 + 表单） | 上下堆叠 |

**适配哲学**：这是"不崩版"级别的基础适配，不是为移动端专门重设计的体验。桌面端仍是主演示场景（投资人/客户会议、PR 分享）。移动端只保证：
- 所有页面能正常渲染（无内容溢出或截断）
- 关键操作可点击
- 内容可读，即便需要横向滚动

       │
